# Dependable Online Payment System: Modelling, Simulation and Verification

This repository contains a dependability assessment and simulation project for an online payment architecture. The system is designed to be fault-tolerant, incorporating redundancy and database checkpoint/recovery mechanisms. The project includes a discrete-event simulator implemented in **MATLAB** and formal validation conducted via **GreatSPN**.

Developed as part of the *Dependable Computer Modelling and Simulation* course.

---

## Architecture Overview

The system models a high-availability online payment infrastructure consisting of two main tiers:

1. **Front-End (FE) Layer (Parallel Redundancy):** 
   * Composed of two redundant modules working in parallel.
   * The front-end subsystem remains operational as long as at least one module is active.
   * A total system failure at this layer occurs *only* when both modules fail concurrently.

2. **Database (DB) Layer (Checkpoint & Recovery):**
   * A single database instance handles payment persistence.
   * **Checkpointing:** The database periodically saves its state to stable storage to limit data loss.
   * **Recovery:** In the event of a database failure, a recovery mechanism is triggered, restoring the DB state to the last valid checkpoint.

---

## Methodology & Project Structure

The project bridges empirical simulation and formal verification to evaluate the system's **Availability**, **Reliability**, and **Mean Time To Failure (MTTF)**.

### 1. MATLAB Discrete-Event Simulator
The core simulator mimics the stochastic behavior of components (Failure and Repair rates) over time using stochastic distributions.
* Models the parallel lifecycle of both Front-End modules.
* Implements the Database lifecycle, integrating periodic checkpoint time-windows and recovery overhead duration.
* Tracks system states to compute steady-state dependability metrics.

### 2. Formal Verification via GreatSPN
To formally validate the logic of the MATLAB simulation, the system was modeled as a **Generalized Stochastic Petri Net (GSPN)** using **GreatSPN**:
* **Places and Tokens:** Represent the structural and operational states of the Front-Ends, Database, Checkpoint triggers, and Active Recovery phases.
* **Timed & Immediate Transitions:** Used to model exponential failure/repair rates, deterministic checkpoint intervals, and immediate structural logic.
* **Validation:** The Markov Chain automatically generated by GreatSPN was analyzed to cross-verify the steady-state probabilities and throughputs against the empirical results obtained from the MATLAB simulation.
